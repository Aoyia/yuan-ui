# 智能体思维链 (CoT) 渲染与人机协同演进 ── 产品需求文档 (PRD)

在大模型推理吞吐率不断攀升、多 Agent 系统逐渐成熟的背景下，智能体（Agent）的思考过程正变得极其复杂且深长。本需求文档旨在从第一性原理出发，重新定义下一代**智能体思维链（Chain of Thought, CoT）渲染与协同机制**，打破传统瀑布流日志的视觉噪音，构建具备“人机共生”特征的可视化交互网络。

---

## 1. 核心痛点与第一性原理推导

### 1.1 传统瀑布日志式思维链的根本瓶颈
1. **屏幕空间爆炸（Screen Space Explode）**：
   在大模型执行多轮循环、数十个工具调用时，传统的线性列表会迅速拉长。这导致真正的“正文输出结果”被挤压出视口之外，用户需要频繁拖拽滚动条，产生极大的认知负荷。
2. **高噪无意义信息（High Noise Index）**：
   Agent 产生的 90% 以上中间态推理（例如微小的语法搜索、文件列表列出等）在执行成功后都会变为低价值的“历史残存信息”。将它们与当下最需要关注的“活跃步骤”放在同等视觉重要级上是低效的。
3. **黑盒指令无法介入（Black-box Execution）**：
   传统的 CoT 仅具有只读（Read-only）的静态展示属性。如果 Agent 陷入死循环或在第 5 步做出了错误的工具假设，用户只能被动等待最终的错误结果，无法在运行中途进行精准的局部决策纠偏。

### 1.2 第一性原理（First Principles）解构
基于人机协同的本质，我们得出以下设计公理：
* **思维链的几何本质是有向无环图（DAG）**：思考不是线性的单行道，而是包含分支调研、快速试错、剪枝放弃、合并回溯的有向网络。
* **信息消费需要焦点聚合（Attention Focus）**：用户的视阈黄金区域只应该用来高亮呈现“当下活跃”的节点；历史已证实的逻辑链应当自动缩入背景，作为可追溯的暗淡辅助线。
* **人机共生要求“时空穿梭干预”（Time-travel Intervention）**：人与 Agent 的协作应该拥有在历史任意决策点“建立平行分支，覆写未来思维”的能力。

## 2. 核心功能与产品需求定义

### 2.1 动态拓扑图渲染画布 (DAG Canvas)
* **有向拓扑连接**：思维路径与工具节点不再是单纯的垂直 Timeline，而是以 BFS 拓扑算法或力导向算法横向排版、从左到右的多列流。
* **分支与合并**：支持节点的多亲关系（`parentIds`）。当一个推理节点产生了两个并行的子工具任务时，画布应清晰地呈现出分叉分支；当多个子任务的返回值融汇给最终决策时，应呈现合并收纳。
* **剪枝灰化 (Pruned Branches)**：对于大模型在中间步骤自我纠正、决定放弃的思考路径（状态标记为 `pruned`），UI 应当进行大幅度调淡并把连线转为虚线，提示用户“此路已被模型自我剪裁”，既能保留思考原貌，又极大地降低了视觉干扰。

### 2.2 渐进式注意力折叠与视口跟随 (Progressive Attention Suppression)
* **同级坍缩机制**：当当前层级（比如 Level 3）诞生了新的 `active` 活跃节点时，该层级之前所有已处于 `complete` 完成态的兄弟节点及其所展开的输入输出参数（IO Panels）应自动缩合为极简的高度。
* **祖先淡入淡出**：上一级（Level 2、Level 1）的祖先大步骤自动暗淡化处理，仅保留淡淡的轮廓和连线，将用户的视线以物理重力感拉拽到当下最新的 Step。
* **流式平滑锚定**：在流式打字追加时，视口具有动态滚动锁定保护，防止折叠导致的屏幕跳闪和视觉丢失。

### 2.3 时空分叉干预拦截闸 (Time-travel Intervention & Fork)
* **全生命周期 Fork**：
  在任何一个已渲染的推理（reasoning）或工具（tool）节点卡片上，均提供一个“时空分叉”干预入口。
* **平行时空分裂**：
  点击 Fork 后，弹出时空纠偏输入框。当用户输入新的纠偏建议或修改指令（例如：“改用 fetch 模块，并且不要读取 test.js”）并提交：
  1. 系统会立即把该节点在时空线上的**所有未来子孙节点**强置为 `pruned` 剪枝状态，表达该段未来已覆灭；
  2. 从该节点右侧长出一条发出高亮呼吸灯流光的新分支，向应用端与大模型重传修改后的上下文，从而实现多分支试错。
* **敏感高危拦截闸 (Interceptor Guard)**：
  产品内置敏感动作拦截规则（如终端写文件、脚本删除、远程网络读写等）。在进入执行前自动拦截并弹框，必须经由用户点击“Approve（授权执行）”或“Reject（驳回修改，启动 Fork）”方可继续推进。

## 3. 全局交互状态机定义 (State Machine)

每个思维或工具节点卡片在生命周期内必须处于以下核心状态之一：

| 状态名 | 视觉表现 | 物理寓意 |
| :--- | :--- | :--- |
| **`pending`** | 灰底、静态半透、无呼吸灯 | 节点已声明但尚未被执行，在列队中排载。 |
| **`active`** | 毛玻璃亮白背景、蓝色发光外阴影、呼吸灯闪烁、贝塞尔连线伴随脉冲粒子流 | 节点正在流式写入（Reasoning Delta 中）或正在后台调用工具。 |
| **`complete`**| 淡白背景、深灰文字、绿色勾标、连线置为平滑细灰色实线 | 节点执行成功，进入非活跃只读历史，参数部分自动折叠。 |
| **`error`** | 浅红发光边框、红色错误图标、红色连线 | 工具执行抛出异常或推理崩溃，流程阻断，等待用户介入。 |
| **`cancelled`**| 置灰偏暗 | 流程被用户或模型强行终止。 |
| **`pruned`** | 大幅度降噪（不透明度 0.3-0.45）、连线转为灰色点状虚线 | 时空分叉或模型自我纠偏导致的废弃分支。鼠标悬浮可显现半透明悬浮卡。 |

## 4. 异常逻辑与边界场景

1. **并行高吞吐率渲染卡顿**：
   若大模型并发产生几十个流节点，且频繁获取 DOM 进行 SVG 画布贝塞尔连线重绘，会导致界面掉帧。产品必须规定最大单次重绘阈值与动画节流周期（例如每 16ms 限制重绘一次）。
2. **极深树与循环依赖**：
   虽然大模型产生的是有向无环图，但需严防因 parentId 配置异常导致图结构死循环。检测到环路（Loop）时必须自动降级为线性时间轴排版并报警。
3. **分叉过载清理 (Garbage Collection)**：
   若用户频繁分叉产生超过 10 条剪枝路径，内存及 SVG 元素过多，画布应根据 LRU 算法，仅保留最近活跃的 3 个分支路径，其余极古老剪枝分支自动隐藏并合并为“省略的剪枝路径”折叠图标。
