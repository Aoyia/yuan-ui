# 对标 Cursor 体验的 Composer 风格 AgentTrace 设计规格说明书

本设计文档旨在将 `AgentTrace` 组件（Timeline 模态）重构并优化为**嵌入聊天流的极简卡片流水线（Composer 风格）**，以提供极其静默、高空间利用率和极致交互细节的开发者体验。

---

## 1. 目标与定位 (Goal & Position)

*   **使用场景：** 嵌入聊天对话流（Chat / Composer UI）的思考过程与工具调用过程展示。
*   **交互原则：**
    *   **低视觉噪音：** 默认高度折叠，不打扰用户的正常聊天流。
    *   **高密度信息：** 原地展开，限高展示（代码差异 Diff、终端控制台输出），使用自定义滚动条，防止聊天框被撑爆。
    *   **丝滑微动效：** 包含打字机流式输出、呼吸灯、状态切换过渡动画。

---

## 2. 架构设计与组件规范

总体上由外层容器、顶部 Header 触发器、内容折叠区以及递归的节点列表组成：

```
[AgentTrace] (外层容器，管理状态与过渡)
  ├── [AgentTraceTrigger] (大容器 Header，负责折叠摘要、全局耗时与计时器)
  └── [AgentTraceContent] (风琴过渡区，高度伸缩动画)
        └── [AgentTraceList] (节点列表递归渲染)
              ├── [ReasoningTraceNode] (极简推理节点)
              ├── [ToolTraceNode] (工具调用卡片) ── 原地展开 ── [TerminalRenderer / DiffRenderer]
              └── [ArtifactTraceNode] (产物气泡卡片)
```

---

## 3. 详细设计规格 (Detailed Specs)

### 3.1. 统一聚合容器与全局状态 (`AgentTrace` & `AgentTraceTrigger`)

*   **数据模型状态 (Global State)：**
    *   `isStreaming`: 表示 Agent 是否正在思考输出中。
    *   `duration`: 整个 Trace 生命周期的总耗时（秒）。
*   **折叠态样式与交互：**
    *   高度限制在 `32px` 左右，背景融入聊天泡中（亮色下：`rgba(0,0,0,0.02)`，暗色下：`rgba(255,255,255,0.02)`），边框为极淡的灰色 `1px`（亮色下：`#f1f5f9`，暗色下：`#27272a`）。
    *   **左侧微型呼吸灯（Status Dot）：**
        *   运行中：淡蓝色小圆点（`#0071e3` / `#2997ff`），配合 `scale` 与 `opacity` 循环变化的呼吸动画。
        *   成功完成：静态柔和绿点（`#34c759` / `#30d158`）。
        *   执行出错/取消：静态暗红点（`#ff3b30` / `#ff453a`）。
    *   **时间计时器（Dynamic Timer）：**
        *   如果 `isStreaming === true`，秒数启动，以 `0.1s` 为步长流式递增（如 `正在思考与运行工具... (2.4s)`），提升交互的真实感。
        *   结束后，显示耗时（如 `已完成思考并执行了 3 个工具 (总耗时 4.2s)`）。
    *   **折叠控制：** 点击整个 Header 区域实现折叠/展开，展开箭头（Chevron）带有 `180deg` 平滑旋转动画。

### 3.2. 极简 Tool 节点卡片 (`ToolTraceNode`)

*   **折叠态（极致压缩）：**
    *   高 `28px`，左右内边距 `8px`。背景亮色 `#f8fafc`，暗色 `#1e1e2e`，圆角 `4px`。
    *   **图标与简写：** 
        *   如果是执行命令，显示微型 `Terminal` 图标，标题为命令内容（如 `运行测试: npm run test`）；如果是修改文件，显示微型 `FileCode` 图标，标题为目标路径（如 `修改 src/App.vue`）。文字超出卡片时尾部自动截断。
        *   图标颜色根据单步状态点亮（运行中为蓝色闪烁，成功为灰色，失败为红色）。
*   **展开态（原地插片式展开）：**
    *   **输入块 (Input Block)：**
        *   如果是纯文本命令或代码，渲染为 `11px` 的只读块。
        *   如果是复杂 JSON，提供一个微型的“参数”折叠标签，点击后才展开，防止初始展开占位过大。
    *   **代码差异渲染 (`DiffRenderer`)：**
        *   高度设定：`max-height: 200px`，超出部分出现内嵌滚动条。
        *   自定义滚动条：宽度 `4px`，无滑轨边框，滚动时 Diff 顶底边缘使用 `linear-gradient` 进行 `4px` 范围的淡出羽化，消解硬边界。
        *   配色方案：删除行采用极淡红 `rgba(255, 59, 48, 0.04)`，增加行采用极淡绿 `rgba(52, 199, 89, 0.04)`，行内字样只做纯文字，不做过度刺眼的反差背景。
    *   **终端日志渲染 (`TerminalRenderer`)：**
        *   高度设定：`max-height: 160px`，等宽字体 `11px`，深灰/碳黑暗护眼背景（亮暗色下统一使用深色黑底控制台），提供真实的 Terminal 氛围。
    *   **人工审批面板 (Approval Panel)：**
        *   当进入 `approval-requested` 状态时，卡片底部平滑滑出按钮行（亮色毛玻璃阴影，暗色淡蓝微光）。
        *   提供 `拒绝` 按钮与 `授权执行` 按钮，点按后触发全局回调。

### 3.3. 极简推理节点 (`ReasoningTraceNode`)

*   **视觉规格：**
    *   左侧是一条细线（与整体时间轴线融合），节点只在 active 时闪烁。
    *   推理文本（summary）采用极淡文字（亮色 `#64748b`，暗色 `#94a3b8`），流式输出时采用淡入过渡，字号 `12px`。
    *   支持 `visibility` 过滤：若为 `redacted` 则展示“已隐藏推理详情”，保障安全合规。

---

## 4. 规格自检 (Spec Self-Check)

1.  **占位符扫描：** 无 "TODO" 或 "待定" 的不确定规范。
2.  **内部一致性：** `AgentTrace` 管理的全局计时与内部 `ToolTraceNode` 单步耗时逻辑统一，且审批机制通过 React/Vue 上下文（Context/Provide）通信。
3.  **范围检查：** 本期规格完全聚焦在 Timeline 模式的 UI 体验与 CSS 微动效优化上。
4.  **模糊性消除：** 明确了即使在亮色模式下，`TerminalRenderer` 依旧使用深色控制台设计，以保证最佳的命令行渲染观感。

---

## 5. 验证与测试计划 (Verification Plan)

### 5.1. 自动化单元测试
*   运行 `npm run test` 以确保基本组件挂载正确。
*   编写单元测试：验证 `AgentTrace.vue` 接收 `isStreaming` 时能够正确触发计时器并向上层派发事件。

### 5.2. 手动与视觉验证
*   启动本地 Vite 演示项目 (`npm run dev` 或使用 Vite Demo 配置)。
*   在不同的亮色/暗色主题（Dark Mode Toggle）下，观察：
    1.  大容器展开和折叠的 Gird 动画是否丝滑。
    2.  `DiffRenderer` 在长代码变更下，内部滚动条和高亮边缘羽化效果是否正确。
    3.  呼吸状态灯在不同状态下的颜色 and 动画周期。
